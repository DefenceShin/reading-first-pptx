# Component System

Use these native PowerPoint components as reusable building blocks. Do not treat them as visual decoration; each component must explain, compare, qualify, or prioritize content.

## Component Selection

- **Conclusion title**: Use on every analytical slide. It should state the answer, not merely name the topic.
- **Kicker chip**: Use for section, source, time period, or analysis lens. Keep it short and factual.
- **Local header**: Use inside a dense body area to label a module. Prefer this over large colored cards.
- **Thin divider**: Use to separate logic groups while keeping the page light.
- **Annotation strip**: Use for implication, boundary, caveat, or method note.
- **Evidence cell**: Use inside matrices or tables for compact fact-plus-meaning statements.
- **Metric callout**: Use for one or two important numbers only. Pair the number with meaning.
- **Risk tag**: Use for severity, dependency, confidence, or implementation condition.
- **Decision note**: Use when the slide should end with a recommendation or action implication.
- **Report header rule**: Use a thin top line plus compact metadata when the slide should read as a consulting/report page.
- **Red anchor geometry**: Use one red circle, arc, arrow, marker, or highlight block to establish the scanning focal point.

## Native PPTX Rules

- Build with editable text boxes, shapes, tables, lines, and charts.
- Prefer PowerPoint charts or editable tables when source data exists.
- Use screenshots only when the screenshot is itself evidence.
- Do not flatten layouts into a single exported image.
- Keep grouped objects easy to edit: group structural shapes together, but leave important text accessible when practical.
- Preserve source data separately when a chart is recreated.

## Density Components

### Comparison Row

Use for A/B choices, version comparisons, vendor/product options, or before/after logic.

Structure:

- left label
- 2-4 comparable attributes
- one implication cell
- optional risk/confidence tag

Avoid unequal paragraphs that make comparison impossible.

### Evidence Stack

Use when multiple facts support one conclusion.

Structure:

- local header
- 3-5 evidence rows
- each row has claim, proof, implication

Do not use a generic bullet list when a claim-proof-implication stack is clearer.

### Analytical Table

Use when the reader must scan many items.

Rules:

- keep row labels short
- use light grid lines
- emphasize only the column or cells that matter
- add a compact takeaway outside the table
- avoid wrapping so much text that row height becomes unstable

### Framework Map

Use for operating models, maturity models, risk systems, or decision frameworks.

Rules:

- show dimensions and relationships, not isolated boxes
- add at least one business meaning, limitation, implementation condition, trade-off, risk, or implication
- keep the map readable without speaker narration

### Chart Interpretation Block

Use on data slides.

Structure:

- chart title or axis labels that preserve the source meaning
- one red or navy emphasis on the key point
- interpretation block explaining the delta, limit, or implication
- source or method note when needed

Do not leave a chart without an interpretation block.

### Consulting Header

Use on dense analytical pages that need report-like continuity.

Structure:

- thin horizontal rule near the top
- concise page title or section label
- optional date/source/page marker
- small logo or mark only when provided by the user

Do not copy prototype watermarks or vendor marks.

### Red Anchor

Use when one region of a dense slide needs immediate visual priority.

Good uses:

- key number
- decisive arrow
- active process step
- highest-risk cell
- main comparison result
- center of a framework map

Avoid using multiple red anchors with equal weight on the same slide.

## Image Components

- Use images at stable aspect ratios and avoid accidental stretching.
- Crop only when the cropped area is not evidence.
- Add short captions that explain why the image matters.
- For screenshots, preserve legibility of the original UI/text.
- If image quality is not sufficient, use a placeholder and mark the risk instead of pretending it is verified.

## Emphasis

Use at most two emphasis devices per slide:

- red text or red tag
- bold title phrase
- navy chip
- highlight line
- metric callout

If everything is emphasized, nothing is emphasized.

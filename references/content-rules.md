# Content Rules

## Density

- High information density is normal.
- Keep one obvious message per slide.
- Use zoning and grouping instead of shrinking every object.
- Make the page readable after the meeting, not only during the presentation.
- Do not trade away substance for spaciousness.
- Keep the top area clean and reserve the center for most of the payload.
- If several modules share one page, make them read as one argument.
- Keep one dominant focal area even on dense pages.
- Make the main conclusion visible within a few seconds.

## Inputs

- rough notes
- half-written outlines
- chat transcripts
- tables and spreadsheets
- screenshots
- reports
- PDFs
- Word files
- markdown
- existing decks

## Transformation

1. identify the real topic and page objective
2. separate facts from interpretation
3. compress repetitive wording
4. rebuild a cleaner structure
5. add missing connective explanation
6. add limits, implications, trade-offs, or risk framing when needed

## Structure Options

- If one source pack supports several plausible stories, generate 2-3 outline or page-structure directions.
- Keep all directions inside the same visual language.
- Recommend one direction and state the trade-off briefly.
- Do not expand this into a multi-style exercise.

## Inference

Allowed:

- turn rough notes into clearer titles
- reorganize bullet lists into tables, matrices, or frameworks
- add missing transition logic between facts
- rewrite fragmented notes into compact analytical prose
- summarize a long source into a denser page structure

Not allowed:

- invent data
- fabricate citations
- make up dates, customers, market shares, or percentages
- claim a business conclusion without support from the provided material
- present assumptions as confirmed facts

## Thin Content

- add clearer issue framing
- add comparison structure
- add boundary conditions
- add implementation conditions
- add likely trade-offs
- add risk notes
- add implication statements

Do not pad with empty motivational or strategic language.

For structural or technical topics, add business meaning, limitation, implementation condition, trade-off, risk, or implication when the source allows it.

## Sample-First Rule

- If density risk, structure risk, or visual risk is high, preview 1-2 key slides before full production.
- Use one dense body slide and one chart or comparison slide when the deck depends on both.
- Use the preview to validate whether the page actually works, not just whether the outline sounds reasonable.

## Warning Signs

Too light:

- it feels like a poster more than a working business slide
- most of the page is empty space around one oversized graphic
- the page cannot stand on its own as a reading document

Too heavy:

- the main conclusion is hard to find within a few seconds
- every region has the same visual weight
- support notes overwhelm the core structure instead of clarifying it

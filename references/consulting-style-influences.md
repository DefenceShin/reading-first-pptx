# Consulting Style Influences

This skill's default visual language is already close to a McKinsey-style working deck. Use these rules only on the default/red-template path defined in `SKILL.md`. Preserve the local color and typography system.

Reference observed: `https://github.com/seulee26/mckinsey-pptx`.

## What To Borrow

- Select a concrete page type before composing.
- Use conclusion titles and short section markers instead of decorative headlines.
- Keep body pages dense, structured, and editable, with the analytical object carrying most of the page.
- Pair charts and tables with interpretation panes, not free-floating visuals.
- Prefer compact callouts, source notes, method notes, and footer/page markers over big decorative blocks.
- Make slide-level decisions defensible: when several structures could fit, choose the one that best matches the evidence, number of items, axis type, and audience use case.
- Use tight margins, thin rules, aligned columns, and local headers to create polish.
- Use a disciplined consulting story arc for longer decks: bottom line or executive summary, issue decomposition, supporting analysis, prioritization or options, roadmap or implications, recommendation, appendix.

## What To Preserve Locally

- Keep the default colors from `style-anchor.md`: Navy `#263F6A` and Crimson/Red `#AF0837`.
- Keep the default fonts from `style-anchor.md`.
- Keep white or near-white body slides as the default canvas.
- Keep the local red-anchor rhythm. Do not replace it with a blue-only McKinsey palette.
- Keep native PowerPoint editability: text boxes, shapes, tables, charts, and lines.

## What To Avoid

- Do not import or depend on the external `mckinsey_pptx` package as the default production path.
- Do not copy logos, watermarks, copyright text, exact slide surfaces, or proprietary-looking artifacts.
- Do not copy the external theme colors when they conflict with the local palette.
- Do not turn every deck into a dark-navy presentation; use high-contrast thesis pages sparingly.
- Do not leave generic placeholder labels in real decks. Replace placeholders with content-specific wording or mark assumptions clearly.

## Practical Translation

- External dark-navy impact pages -> local bottom-line thesis pages using the existing navy/red system.
- External template catalog discipline -> local pattern/component rationale in the deck plan.
- External chart templates -> local chart-with-interpretation pages with editable charts and concise takeaway panes.
- External matrix templates -> local comparison, prioritization, or portfolio pages with clear axes and one red focal area.
- External roadmap/process templates -> local roadmap/workplan pages with dependency and risk strips.
- External executive-summary templates -> local executive-summary dense pages with 2-4 takeaways and evidence bullets.

Use the influence as reusable grammar, not as a literal visual clone.

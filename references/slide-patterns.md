# Slide Patterns

Choose a pattern before placing objects. A pattern is a reasoning structure first and a layout second.

## 1. Issue Explainer

- conclusion-style title
- one central framework, flow, or annotated diagram
- side notes that explain implications or boundaries
- a short takeaway strip

Use when the audience needs to understand why a topic matters before seeing options or actions.

If the issue has a clear decomposition, use an issue-tree structure with branches and leaf implications instead of a loose concept diagram.

## 2. Comparison Matrix

- conclusion-style title
- matrix, ranked table, or structured comparison grid
- one short note per key contrast
- a bottom-line implication

Use when the same criteria apply across options. Keep criteria parallel.

Avoid turning each option into a large standalone card unless the comparison logic truly differs by module.

## 3. Process Or Sequence Page

- title with the overall message
- central process chain or staged structure
- compact notes under or beside each stage
- one risk or condition strip

Use when time, ownership, dependency, or state transition is central.

## 4. Chart With Interpretation

- title with the core numerical story
- chart or table as the main body
- red emphasis on one or two key values only
- a note block explaining meaning, limits, or comparison
- navy chart framing, compact labels, and visible data callouts when useful

Use when the slide would be weaker without the data shape.

Do not place a chart on the page without interpretation.

For executive-facing chart pages, prefer a main chart plus a right-side or bottom takeaway pane. Use one red emphasis point and keep all other series restrained.

## 5. Dense Table Page

- strong title
- compact table with disciplined hierarchy
- local chips or emphasis cells
- takeaway or warning note
- navy table headers, light gray support fields, and red key cells when the contrast is meaningful

Use when the reader must scan many items after the meeting.

For KPI/status work, use an assessment-table structure with target, actual, status, and implication fields when those dimensions exist.

## 6. Risk / Boundary Page

- title that states the boundary clearly
- grouped risk or condition modules
- supporting evidence or scenarios
- one implication strip for management or implementation meaning

Use when the right answer depends on constraints, dependencies, or confidence.

## 7. Recommendation Page

- decisive title
- primary recommendation block
- 2-4 supporting reasons or conditions
- one short next-step strip

Use when the deck has enough evidence to support an action.

## 8. Appendix Page

- plain title
- compact, factual body
- less visual drama than the main deck
- keep the same font and color logic

Use for detail retention, not for unresolved main-argument content.

## 9. Source Synthesis Page

- title that states the synthesis conclusion
- grouped evidence from multiple source types
- small source labels or confidence tags
- one implication or gap note

Use when notes, reports, screenshots, and tables must be reconciled into one reading page.

## 10. Decision Tree / Option Path

- title that states the decision condition
- left-to-right or top-to-bottom branching structure
- compact rule labels
- endpoint implications or recommended choices

Use when the reader needs to decide based on conditions rather than compare fixed options.

## 10A. Prioritization / Portfolio Matrix

- title that states the decision implication
- 2x2 or 3x3 matrix with clear axis meaning
- items plotted or placed into cells
- one red focal zone, marker, or recommended cluster
- takeaway pane explaining what to do next

Use for initiative prioritization, market attractiveness, capability gaps, portfolio position, or risk/impact evaluation. Do not use when the data is a continuous scatterplot that needs exact x/y values; use a chart-with-interpretation page instead.

## 10B. Logical Architecture / System Map

- title that states the system meaning
- editable node-link structure with navy structural nodes
- red focal circle, arrow, or key node when one element matters most
- compact notes explaining business meaning, risk, or implementation condition

Use for frameworks, operating models, product systems, data flows, or solution architecture. Keep it business-readable rather than technical-drawing-like.

## 11. Before / After Transformation

- title that states what changed
- two aligned zones with the same substructure
- highlighted deltas
- implication strip

Use for process redesign, product change, policy change, or operating model shifts.

## 12. Executive Summary Dense Page

- conclusion title
- 3-5 grouped takeaways
- one evidence or metric anchor per takeaway
- bottom-line implication

Use at the front of a working deck when the audience needs the answer before the detail.

## 13. Roadmap / Workplan

- title that states the delivery logic or sequencing conclusion
- horizontal timeline, phase chevrons, wave plan, or milestone table
- compact notes for each phase or milestone
- one dependency, risk, or decision strip

Use when timing, sequencing, and ownership are part of the argument. Keep it closer to a management workplan than a decorative timeline.

## 14. Bottom-Line Impact / Section Thesis

- one decisive sentence
- optional eyebrow, section marker, or small context line
- restrained high-contrast field or concentrated navy region
- no bullet clutter

Use sparingly for major section resets, core findings, or closing arguments. Do not use for ordinary content pages.

## Page Rhythm

For decks longer than 8 slides, avoid repeating the same structure too many times.

Good rhythm examples:

- summary -> issue explainer -> comparison -> chart -> recommendation -> appendix
- context -> source synthesis -> process -> risk boundary -> decision tree -> recommendation
- issue -> chart -> dense table -> before/after -> recommendation -> appendix
- bottom-line thesis -> executive summary -> issue tree -> chart with takeaway -> prioritization matrix -> roadmap -> recommendation -> appendix

Do not add decorative divider pages unless they help the reader reset between major sections.
